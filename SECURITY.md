# 安全政策

## 支持的版本

| 版本 | 是否接受安全修复 |
|------|----------------|
| 0.1.x（当前） | ✅ 是 |

## 报告漏洞

**请勿通过公开 GitHub Issue 报告安全漏洞。**

如果你发现了安全漏洞，请通过以下方式私下披露：

**邮箱**：byteforce@qq.com

请在邮件中包含以下信息：

- 漏洞类型（如缓冲区溢出、SQL 注入、跨站脚本等）
- 受影响的源文件路径（含完整路径）
- 触发漏洞所需的特殊配置
- 复现漏洞的步骤
- 概念验证代码或攻击场景（如有）
- 漏洞可能造成的影响，包括攻击者如何利用该漏洞

我们将在 **5 个工作日内**确认收到你的报告，并在 **30 天内**提供修复方案或缓解措施的详细说明。

## 披露政策

在漏洞修复发布后，我们将在 `CHANGELOG.md` 中公开披露漏洞详情，并在 GitHub Security Advisories 中发布 CVE（如适用）。

## 范围

- `crates/coord-server`：gRPC 服务端、HTTP 控制面
- `crates/coord-ctl`：命令行工具
- `sdk/`：各语言 SDK（Rust / Java / Go）
- 安全控制面（Seal/Unseal、AppRole 认证、Token 鉴权、Shamir 拆分、Transit 加密、PKI）

以下内容**不在**范围内：

- 依赖上游库自身的漏洞（请直接上报上游项目）
- DoS 攻击（尤其针对开发模式服务端）
- 社会工程学攻击

## 首选语言

我们接受中文或英文的漏洞报告。
